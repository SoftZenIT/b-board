# BJ-Keyboard Architecture

## Overview

BJ-Keyboard (`b-board`) is a framework-agnostic virtual keyboard library built with TypeScript. It provides a composable keyboard engine with multi-language support, a composition layer for tonal/nasal/dead-key input, and UI renderers for desktop and mobile.

## Module Boundaries

```
src/
├── public/         ← PUBLIC API (only import from here)
│   └── index.ts   ← Single consumer entry point
│
├── core/           ← INTERNAL: state machine, runtime engine
├── data/           ← INTERNAL: loaders, validators, type definitions
├── composition/    ← INTERNAL: tone/nasal/dead-key composition logic
├── ui/             ← INTERNAL: Lit Web Component renderers (framework-agnostic)
├── adapters/       ← INTERNAL: host target integrations (input, textarea, contenteditable)
├── components/     ← INTERNAL: Vue component wrappers
└── utils/          ← INTERNAL: shared helpers
```

## Public vs Internal

| Layer              | Visibility | Description                           |
| ------------------ | ---------- | ------------------------------------- |
| `src/public/`      | **Public** | Everything a consumer can import      |
| `src/core/`        | Internal   | State machine, keyboard engine        |
| `src/data/`        | Internal   | Type definitions, loaders, validators |
| `src/composition/` | Internal   | Input composition logic               |
| `src/ui/`          | Internal   | Lit Web Component renderers           |
| `src/adapters/`    | Internal   | Host target adapters                  |
| `src/components/`  | Internal   | Vue wrappers for dev environment only |
| `src/utils/`       | Internal   | Shared utility functions              |

> **Rendering stack note:** `src/ui/` uses **Lit** (Web Components) so the keyboard renderer is framework-agnostic. `vue` and `src/components/` exist only for the dev/demo environment. The published library ships Lit-based custom elements.

**Rule**: Consumers MUST only import from `b-board` (which resolves to `src/public/index.ts`). Cross-module imports within `src/` follow a strict dependency direction:

```
ui → core → data
adapters → core
composition → core → data
public → (re-exports from ui, core, adapters)
```

## Build Output

The Vite library build produces:

- `dist/bboard.es.js` — ES Module (for bundlers)
- `dist/bboard.umd.js` — UMD (for CDN / legacy)
- `dist/index.d.ts` — TypeScript declarations (generated by `vite-plugin-dts`)
